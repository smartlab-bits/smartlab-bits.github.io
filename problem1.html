PSUEDO - CODE

X[1..N] is a set of positive integers. (array)

PROCEDURE( X[1..N] ):
     Step[1] -
         j = 1
         k = N - 1
         m = X[N]
         goto Step[2]

     Step[2] -
         if k is 0
             terminate
         else
             goto Step[3]

     Step[3] -
         if X[k] <= m
             goto Step[5]
         else
             goto Step[4]

     Step[4] -
         j = k
         m = X[k]
         goto Step[5]

     Step[5] -
         k = k-1
         goto Step[2]

Answer the following questions with regard to the above pseudo-code

  1. Justify why this procedure will always terminate in finite number of steps.
  2. What is the output produced by the above procedure?
  3. Write down the number of times each step will be executed as a function of N.
  4. Provide a test case for which the algorithm will execute for: (a)Maximum number of steps (b)Minimum number of steps
  5. Assuming that the input X[1..N] is some permutation of the set {1..N}, find out the average number of times Step 4 is executed. [Hint: Each permutation of {1..N} is equally probable]
  6. Does the answer for Q5 change, if X[1..N] is allowed to take any N distinct integer values ?
  7. Implement the code for the above given procedure in your preferred programming language.
